## ğŸ§  Intuition
The solution uses:
1. **Three-Way Partitioning**: Divides array into three sections
2. **Three Pointers**:
   - `i` tracks end of 0s section
   - `j` tracks start of 2s section
   - `k` scans the array

Key Insights:
- All elements before `i` are 0s
- All elements after `j` are 2s
- Elements between `i` and `k` are 1s
- Elements after `k` are unexplored

## âš™ï¸ Approach
1. **Initialization**:
   - `i=0` (start of array)
   - `j=n-1` (end of array)
   - `k=0` (current element)
2. **Partitioning**:
   - When `nums[k]==0`: Swap with `nums[i]` and move both `i` and `k`
   - When `nums[k]==2`: Swap with `nums[j]` and move `j` back
   - When `nums[k]==1`: Just move `k` forward
3. **Termination**:
   - Stop when `k` crosses `j`

## âœ… Optimized Solution
```java
class Solution {
    public void sortColors(int[] nums) {
        int low = 0, mid = 0, high = nums.length - 1;
        
        while (mid <= high) {
            switch (nums[mid]) {
                case 0: 
                    swap(nums, low++, mid++);
                    break;
                case 1:
                    mid++;
                    break;
                case 2:
                    swap(nums, mid, high--);
                    break;
            }
        }
    }
    
    private void swap(int[] nums, int i, int j) {
        int temp = nums[i];
        nums[i] = nums[j];
        nums[j] = temp;
    }
}
```

## Alternative Implementation (Counting Sort)
```java
class Solution {
    public void sortColors(int[] nums) {
        int[] count = new int[3];
        for (int num : nums) count[num]++;
        
        int index = 0;
        for (int color = 0; color < 3; color++) {
            while (count[color]-- > 0) {
                nums[index++] = color;
            }
        }
    }
}
```

## Historical Context
