# 📌 Minimum Number of Seconds to Make Mountain Height Zero

## 📜 Problem Statement

**Link:**  
[https://leetcode.com/problems/minimum-number-of-seconds-to-make-mountain-height-zero/description/?envType=daily-question&envId=2026-03-13](https://leetcode.com/problems/minimum-number-of-seconds-to-make-mountain-height-zero/description/?envType=daily-question&envId=2026-03-13)

You are given:
- `mountainHeight`: initial height of a mountain (integer)
- `workerTimes`: array, workerTimes[i] = number of seconds for worker i to reduce the mountain height by 1 in the first second (and increasing by 1 for each subsequent reduction: 1, 2, ... units at a time).

You can assign each worker any amount of mountain to reduce.  
Worker `i` can reduce at most `hi` units if they have enough time to do so.  
Find **the minimum number of seconds** needed to reduce the mountain's height to zero.

---

# 🧠 Understanding the Problem

- Each worker removes 1, 2, ..., hi using time `workerTimes[i] * (1 + 2 + ... + hi)`
- For a given time, each worker can reduce some height. Have to sum over all workers whether total height can be reached.
- Need to find the minimal `T` such that, with all workers, sum of possible heights cleared in `T` seconds ≥ mountainHeight.

---

# 💻 Your Solution

### Approach:

- Use binary search for minimal total time `T` (outer).
- For a given `T`, for each worker, do binary search on max possible units (`hi`) they can clear within time `T`.
- For each possible `T` in [0, 1e16], sum all the height each worker can reduce ≤ `T`.
- If total reductions cover mountainHeight, try smaller `T`. Else, try larger.

### Code (Cleaned):

```java []
class Solution {
    public long minNumberOfSeconds(int mountainHeight, int[] workerTimes) {
        Arrays.sort(workerTimes);
        long lowTime = 0, highTime = (long) 1e16, minRequiredTime = highTime;

        while (lowTime <= highTime) {
            long midTime = lowTime + (highTime - lowTime) / 2;
            long totalHeightReduced = 0;

            for (int baseTime : workerTimes) {
                // Max h s.t. baseTime * (h * (h+1)/2) <= midTime
                long lowHeight = 0, highHeight = mountainHeight, maxWorkerHeight = 0;
                while (lowHeight <= highHeight) {
                    long midHeight = lowHeight + (highHeight - lowHeight) / 2;
                    long timeNeeded = (long) baseTime * midHeight * (midHeight + 1) / 2;

                    if (timeNeeded <= midTime) {
                        maxWorkerHeight = midHeight;
                        lowHeight = midHeight + 1;
                    } else {
                        highHeight = midHeight - 1;
                    }
                }
                totalHeightReduced += maxWorkerHeight;
                if (totalHeightReduced >= mountainHeight) break;
            }

            if (totalHeightReduced >= mountainHeight) {
                minRequiredTime = midTime;
                highTime = midTime - 1;
            } else {
                lowTime = midTime + 1;
            }
        }
        return minRequiredTime;
    }
}
```

---

# ⏱️ Complexity Analysis

- Outer binary search: O(log(maxTime)) (~64)
- For each T, do for each worker: inner binary search O(log(mountainHeight))
- So: overall O(W·log(maxTime)·log(mountainHeight)) (W = number of workers)

---

# 💡 Key Insight

- Double binary search: outer for time, inner for per-worker max possible height in given time.
- Simulation not feasible due to large T/mountainHeight.

# 🧠 Takeaway

- For resource allocation problems with increasing cost per unit, always check if a double binary search is appropriate.
- Casting multiplication intermediate values to long avoids overflow.
