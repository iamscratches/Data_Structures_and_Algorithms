# 📌 Find Kth Bit in Nth Binary String

## 📜 Problem Statement

**Link:**  
[https://leetcode.com/problems/find-kth-bit-in-nth-binary-string/description/?envType=daily-question&envId=2026-03-03](https://leetcode.com/problems/find-kth-bit-in-nth-binary-string/description/?envType=daily-question&envId=2026-03-03)

Given two integers `n` and `k`, return the kth bit (1-indexed) in the nth binary string `Sₙ` defined recursively:

- S₁ = "0"
- For n > 1: Sₙ = Sₙ₋₁ + "1" + reverse(invert(Sₙ₋₁))

where reverse(x) is x reversed, and invert(x) flips every bit (0->1, 1->0).

**Example:**  
Input: `n = 3, k = 1`  
S₁ = "0"  
S₂ = "011"  
S₃ = "0111001"  
Output: `'0'` (the 1st bit of S₃)

---

# 🧠 Understanding the Problem

- The sequence is recursively built. Every string doubles in length and is formed by the previous string, a '1', and the reversed/inverted previous string.
- For `Sₙ`, length is (2^n) - 1.
- Generating the entire string for large n is infeasible (exponential growth).

---

# 💻 Your Solution

### Approach:

- Precompute all strings up to n=20 and store in a list.
- To answer queries, just index into the string at (k-1).

### Code (Cleaned):

```java []
class Solution {
    private static List<String> Sn = findTill20();
    public char findKthBit(int n, int k) {
        return Sn.get(n-1).charAt(k-1);
    }

    private static List<String> findTill20(){
        StringBuilder sb = new StringBuilder("0");
        List<String> res = new ArrayList<>();
        res.add(sb.toString());

        for(int i = 0; i < 20; i++){
            sb.append("1").append(revertInvert(res.get(res.size()-1)));
            res.add(sb.toString());
        }
        return res;
    }

    private static String revertInvert(String s){
        StringBuilder sb = new StringBuilder();
        for(int i = 0; i < s.length(); i++){
            sb.append(s.charAt(i) == '1' ? '0' : '1');
        }
        return sb.reverse().toString();
    }
}
```

---

# ⏱️ Complexity Analysis

- Precompute: O(2^20) (done once, unsafe for n > 20)
- Query: O(1), since it's just a string lookup

---

# 🔎 Can We Do Better?

Yes — there's an O(log n) recursive solution without ever building Sₙ:

- Key observation: The middle character is always '1'.
- For k < mid, the answer is just in Sₙ₋₁.
- For k > mid, mirror position, invert answer recursively.
- Only recursion and position math is needed — no string construction.

### Optimized O(log n) Solution

```java []
class Solution {
    public char findKthBit(int n, int k) {
        if (n == 1) return '0';
        int len = (1 << n) - 1, mid = (len + 1) / 2;
        if (k == mid) return '1';
        if (k < mid) return findKthBit(n - 1, k);
        // Mirror position from the other half and invert bit
        char c = findKthBit(n - 1, mid - (k - mid));
        return c == '0' ? '1' : '0';
    }
}
```

---

# 💡 Key Insight

- Sₙ always has a symmetry around its middle '1'.
- Use recursion and bit inversion to find kth bit efficiently.

# 🧠 Takeaway

- Recursively defined strings can often be solved by position and symmetry, not brute construction.
- Always check for O(1) index+recursion solutions to string construction challenges.
