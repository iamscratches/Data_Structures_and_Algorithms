# 📌 Complement of Base 10 Integer

## 📜 Problem Statement

**Link:**  
[https://leetcode.com/problems/complement-of-base-10-integer/description/?envType=daily-question&envId=2026-03-11](https://leetcode.com/problems/complement-of-base-10-integer/description/?envType=daily-question&envId=2026-03-11)

Given a non-negative integer `n`,  
Return the complement of its binary representation as a decimal integer.

- The complement is formed by flipping every bit (0 → 1, 1 → 0) in the binary representation.

**Example:**  
Input: `n = 5` (binary `"101"`)  
Output: `2` (complement is `010`)

---

# 🧠 Understanding the Problem

- The binary representation of `n` is considered without leading zeros.
- The complement flips *every* bit up to the highest one present in `n`.
- If `n` is 0, complement must be 1.

---

# 💻 Your Solution

### Approach:

- Edge case: if `n == 0`, return 1.
- Extract individual bits, flip each, and construct the result bitwise in reversed order.
- Reconstruct the integer from the flipped bits.

### Code (Cleaned):

```java []
class Solution {
    public int bitwiseComplement(int n) {
        if (n == 0) {
            return 1;
        }
        List<Integer> bits = new ArrayList<>();
        while (n > 0) {
            if ((n & 1) == 1) {
                bits.add(0);
            } else {
                bits.add(1);
            }
            n = n >> 1;
        }
        int res = 0;
        for (int i = bits.size() - 1; i >= 0; i--) {
            res = res << 1;
            res = res | bits.get(i);
        }
        return res;
    }
}
```

---

# ⏱️ Complexity Analysis

- Time: O(log n) — number of bits in n.
- Space: O(log n) for the bits list.
- Efficient, no built-in tricks required.

---

# 🔎 Can We Do Better?

Yes — using a bitmask:

- Construct a mask of all 1's with the same length as n's binary.
- Then do `~n & mask`.

### Optimized Solution

```java []
class Solution {
    public int bitwiseComplement(int n) {
        if (n == 0) return 1;
        int mask = 0, tmp = n;
        while (tmp > 0) {
            mask = (mask << 1) | 1;
            tmp >>= 1;
        }
        return ~n & mask;
    }
}
```

---

# 💡 Key Insight

- The complement should only flip bits up to the most significant set bit.
- Use a mask to zero out higher unwanted bits after bitwise not.

---

# 🔥 Comparison

| Approach   | Time     | Space     | Notes                  |
|------------|----------|-----------|------------------------|
| Bit flip   | O(log n) | O(log n)  | Explicit, general      |
| Bit mask   | O(log n) | O(1)      | Compact, bitwise trick |

---

# 🧠 Takeaway

- For binary complements (or related flips), bitmasking is concise and less error-prone.
- Always account for leading zeros and edge cases (`n=0`).
