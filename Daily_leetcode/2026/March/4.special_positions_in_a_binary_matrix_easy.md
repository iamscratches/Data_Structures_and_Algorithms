# 📌 Special Positions in a Binary Matrix

## 📜 Problem Statement

**Link:**  
[https://leetcode.com/problems/special-positions-in-a-binary-matrix/description/?envType=daily-question&envId=2026-03-04](https://leetcode.com/problems/special-positions-in-a-binary-matrix/description/?envType=daily-question&envId=2026-03-04)

Given an m x n binary matrix `mat`, return the number of "special positions" in the matrix.  
A position (i, j) is called special if `mat[i][j] == 1` and all other elements in the same row and column are 0.

**Example:**  
Input:  
```
mat = [[1,0,0],
       [0,0,1],
       [1,0,0]]
```
Output: `1`

---

# 🧠 Understanding the Problem

- A special position is a unique '1' in its row and column.
- Need to check all rows and columns to ensure uniqueness at each potential '1'.

---

# 💻 Your Solution

### Approach:

1. Precompute the number of '1's in each row and each column.
2. For each cell:
   - If `mat[i][j] == 1`, check if count of '1's in its row and its column is 1.
   - If both true, increment the result.

### Code (Cleaned):

```java []
class Solution {
    public int numSpecial(int[][] mat) {
        int[] rowOnes = new int[mat.length];
        int[] colOnes = new int[mat[0].length];

        for (int i = 0; i < mat.length; i++) {
            for (int j = 0; j < mat[0].length; j++) {
                if (mat[i][j] == 1) {
                    rowOnes[i]++;
                    colOnes[j]++;
                }
            }
        }

        int count = 0;
        for (int i = 0; i < mat.length; i++) {
            for (int j = 0; j < mat[0].length; j++) {
                if (rowOnes[i] == 1 && colOnes[j] == 1 && mat[i][j] == 1) {
                    count++;
                }
            }
        }
        return count;
    }
}
```

---

# ⏱️ Complexity Analysis

- First pass (count rows & columns): O(m·n)
- Second pass (check each position): O(m·n)
- **Overall:** O(m·n), space O(m + n)

---

# 🔎 Can We Do Better?

- For general m x n, O(m·n) is optimal because every element may need checking.
- Space usage is already minimal: just two arrays for counts.
- No need for advanced data structures, and matrix size can be up to 10^4, so this approach is efficient and simple.

---

# 💡 Key Insight

- By precomputing row and column '1' frequencies, each special position can be verified in O(1).
- No nested brute-force search is required.

---

# 🔥 Comparison

| Approach          | Time      | Space      | Notes                         |
|-------------------|-----------|------------|-------------------------------|
| Count arrays      | O(m·n)    | O(m+n)     | Efficient, scalable           |
| Brute-force scan  | O(m·n·(m+n)) | O(1)   | Not efficient for big input   |

---

# 🧠 Takeaway

- Frequency maps (counts) are a great way to quickly reduce search space in matrix problems.
- Always look for a way to turn O(m·n·x) scans into a couple of O(m·n) passes with precomputed statistics.
