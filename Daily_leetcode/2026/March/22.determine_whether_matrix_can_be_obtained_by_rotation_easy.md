# 📌 Determine Whether Matrix Can Be Obtained By Rotation

## 📜 Problem Statement

**Link:**  
[https://leetcode.com/problems/determine-whether-matrix-can-be-obtained-by-rotation/description/?envType=daily-question&envId=2026-03-22](https://leetcode.com/problems/determine-whether-matrix-can-be-obtained-by-rotation/description/?envType=daily-question&envId=2026-03-22)

You are given two `n x n` binary matrices `mat` and `target`.

In one operation you can **rotate `mat` 90 degrees clockwise**.

Return `true` if `mat` can be rotated to become exactly equal to `target` (in 0, 90, 180, or 270 degrees), otherwise return `false`.

---

# 🧠 Understanding the Problem

- We can rotate `mat` 0, 1, 2, or 3 times by 90 degrees clockwise.
- After each rotation, check if `mat` equals `target`.
- If any rotation matches, answer is `true`.

Rotation rule for 90° clockwise on `n x n`:
- The element at `(i, j)` in original goes to `(j, n-1-i)` or equivalently `res[n-1-j][i] = mat[i][j]` depending on how you construct.

---

# 💻 Your Solution

### Approach:

1. First compare `mat` with `target` without rotation.
2. If equal, return `true`.
3. Repeat up to 3 times:
   - Rotate `mat` 90 degrees clockwise.
   - Compare with `target`.
4. If any comparison passes, return `true`, else `false` after 3 rotations.

### Code (Cleaned):

```java []
class Solution {
    public boolean findRotation(int[][] mat, int[][] target) {
        if (compare(mat, target)) return true;
        int count = 3;
        while (count > 0) {
            mat = rotate(mat);
            if (compare(mat, target)) return true;
            count--;
        }
        return false;
    }

    private boolean compare(int[][] mat, int[][] target) {
        int n = mat.length;
        for (int i = 0; i < n; i++)
            for (int j = 0; j < n; j++)
                if (mat[i][j] != target[i][j]) return false;
        return true;
    }

    private int[][] rotate(int[][] mat) {
        int n = mat.length;
        int[][] res = new int[n][n];
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                res[n - 1 - j][i] = mat[i][j];
            }
        }
        return res;
    }
}
```

---

# ⏱️ Complexity Analysis

- Comparing matrices: O(n²)
- Rotating matrix once: O(n²)
- Up to 3 rotations: O(3·n²) = O(n²)

Overall:

```
Time  → O(n²)
Space → O(n²) for rotated copy
```

---

# 💡 Key Insight

- At most four distinct rotations (0°, 90°, 180°, 270°) before matrix repeats.
- Simple brute-force over these four cases is optimal and easy to implement.
