# 📌 Largest Submatrix With Rearrangements

## 📜 Problem Statement

**Link:**  
[https://leetcode.com/problems/largest-submatrix-with-rearrangements/description/?envType=daily-question&envId=2026-03-17](https://leetcode.com/problems/largest-submatrix-with-rearrangements/description/?envType=daily-question&envId=2026-03-17)

Given a binary matrix, you may rearrange columns as you like **independently for each row**.  
Find the area of the largest submatrix consisting entirely of 1's after any rearrangement of columns.

---

# 🧠 Understanding the Problem

- Can freely permute columns for each row — need maximal rectangle of all 1's.
- Stack the rows: for each row, count vertical runs of 1's up to that point.
- For each row: sort column heights in non-increasing order.
- Largest possible rectangle for a row = max over all columns (height × width for rightward tail).

---

# 💻 Your Solution

### Approach:

1. For every row except the first, add to matrix[i][j] the value from the row above if matrix[i][j] == 1 (gets the histogram height).
2. For each row:
   - Sort the heights within the row (columns can be rearranged).
   - For each possible width (from sorted array), compute height × width, update max.

### Code (Cleaned):

```java []
class Solution {
    public int largestSubmatrix(int[][] matrix) {
        int m = matrix.length, n = matrix[0].length, res = 0;
        for (int i = 1; i < m; i++)
            for (int j = 0; j < n; j++)
                if (matrix[i][j] == 1)
                    matrix[i][j] += matrix[i - 1][j];
        for (int i = 0; i < m; i++) {
            Arrays.sort(matrix[i]);
            for (int j = 0; j < n; j++)
                res = Math.max(res, matrix[i][j] * (n - j));
        }
        return res;
    }
}
```

---

# ⏱️ Complexity Analysis

- O(m·n·log n) — each row sort dominates
- Space: O(1) extra (in-place on matrix)

---

# 💡 Key Insight

- Sorting each row after heightifying columns makes maximal rectangles easy to compute.
- Tallest/shortest rectangles after column rearrangement can always be grouped at will.

---

# 🧠 Takeaway

- For matrix/rectangle 1's with column rearrangement, histogram method + row-wise sort is optimal.
- Stack runs, sort, and slide!
