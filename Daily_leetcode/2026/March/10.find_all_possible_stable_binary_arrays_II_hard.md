# 📌 Find All Possible Stable Binary Arrays II

## 📜 Problem Statement

**Link:**  
[https://leetcode.com/problems/find-all-possible-stable-binary-arrays-ii/description/?envType=daily-question&envId=2026-03-10](https://leetcode.com/problems/find-all-possible-stable-binary-arrays-ii/description/?envType=daily-question&envId=2026-03-10)

Given `zero` zeros and `one` ones, and a positive integer `limit`,  
Return the number of different binary arrays of length `zero + one` so that:
- Any segment of consecutive zeros has length at most `limit`.
- Any segment of consecutive ones has length at most `limit`.

**Modulo:** return the answer modulo 10⁹+7.

**Example:**  
Input: `zero = 2, one = 2, limit = 1`  
Output: `6`

---

# 🧠 Understanding the Problem

- Place zeros and ones such that no run of 0s or 1s exceeds `limit`.
- Find number of such binary arrays (order matters).
- DP: track remaining 0s/1s and how last block was arranged.

---

# 💻 Your Solution

### Approach:

- Use two 2D DP arrays:
  - `dp0[o][z]`: Ways to build arrays with `o` ones and `z` zeros left, ending with a 0 run.
  - `dp1[o][z]`: Ways to build arrays with `o` ones and `z` zeros left, ending with a 1 run.
- Base cases:
  - When only ones left (<=limit): end with run of ones.
  - When only zeros left (<=limit): end with run of zeros.
- For each state (`o` ones left, `z` zeros left):
  - To place a 0: can append 0 to all ways that previously ended with either 0 or 1, as long as the zero-run is within limit (and subtract overlimit).
  - Symmetrically for placing a 1.
- In each DP cell, subtract overlimit arrangements to maintain the `limit` constraint.

### Code (Cleaned):

```java []
class Solution {
    public int numberOfStableArrays(int zero, int one, int limit) {
        int MOD = (int)1e9 + 7;
        int[][] dp0 = new int[one + 1][zero + 1];
        int[][] dp1 = new int[one + 1][zero + 1];

        for (int i = 1; i <= Math.min(one, limit); i++) {
            dp1[i][0] = 1;
        }
        for (int j = 1; j <= Math.min(zero, limit); j++) {
            dp0[0][j] = 1;
        }

        for (int o = 1; o <= one; o++) {
            for (int z = 1; z <= zero; z++) {
                dp0[o][z] = dp1[o][z - 1];
                dp0[o][z] = (dp0[o][z] + dp0[o][z - 1]) % MOD;
                if (z - limit - 1 >= 0) {
                    dp0[o][z] = (dp0[o][z] - dp1[o][z - limit - 1] + MOD) % MOD;
                }

                dp1[o][z] = dp0[o - 1][z];
                dp1[o][z] = (dp1[o][z] + dp1[o - 1][z]) % MOD;
                if (o - limit - 1 >= 0) {
                    dp1[o][z] = (dp1[o][z] - dp0[o - limit - 1][z] + MOD) % MOD;
                }
            }
        }
        return (dp0[one][zero] + dp1[one][zero]) % MOD;
    }
}
```

---

# ⏱️ Complexity Analysis

- Time: O(one·zero)
- Space: O(one·zero)
- Efficient for moderate input sizes (usually ≤ 200)

---

# 💡 Key Insight

- By reducing DP state to 2D arrays and careful inclusion-exclusion by subtracting overlimit, this approach is faster and less memory-hungry than full 3D DP.
- Use transitions based on how the last run was built and subtract cases where run size exceeds `limit`.

---

# 🔥 Comparison

| Approach          | Time        | Space        | Notes                  |
|-------------------|-------------|--------------|------------------------|
| 2D DP (ends with) | O(o·z)      | O(o·z)       | Efficient, optimal     |
| 3D DP             | O(o·z·2)    | O(o·z·2)     | Clearer but bulkier    |

---

# 🧠 Takeaway

- Refined DP states can drastically reduce time and space, especially for large constraints.
- For combinatorial array arrangements with segment limits, dimensionality reduction is key.
