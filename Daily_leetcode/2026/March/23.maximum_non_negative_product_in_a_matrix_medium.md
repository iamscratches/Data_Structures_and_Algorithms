# 📌 Maximum Non Negative Product in a Matrix

## 📜 Problem Statement

**Link:**  
[https://leetcode.com/problems/maximum-non-negative-product-in-a-matrix/description/?envType=daily-question&envId=2026-03-23](https://leetcode.com/problems/maximum-non-negative-product-in-a-matrix/description/?envType=daily-question&envId=2026-03-23)

You are given an `m x n` matrix `grid` consisting of integers.

- Starting from the top-left cell `(0,0)`, you can only move **right** or **down** to reach the bottom-right cell `(m-1,n-1)`.
- For each path, compute the **product** of all numbers along the path.

Return the **maximum non-negative product** modulo `10⁹ + 7`.  
If **all** path products are negative, return `-1`.

---

# 🧠 Key Observations

- Values in grid can be positive, negative, or zero.
- The product along a path can:
  - grow very large (must use `long`),
  - flip sign whenever multiplied by a negative.
- We cannot just track the maximum product:
  - A very small (negative) product can become maximal when multiplied by another negative later.
- So we need to track **both**:
  - the maximum product up to cell `(i,j)`,
  - the minimum product up to cell `(i,j)`.

This is a classic **DP with max and min** at each cell.

---

# 💻 Your Solution

### DP State

Let:

- `mx[i][j]` = maximum product to reach cell `(i, j)` from `(0,0)`  
- `mn[i][j]` = minimum product to reach cell `(0,0)` → `(i,j)`

Transition:

- Coming from top `(i-1,j)` or left `(i,j-1)`:
  - Multiply both `mx` and `mn` from those cells with the current cell `grid[i][j]`.
  - The new `mx[i][j]` is the maximum of those 4 results.
  - The new `mn[i][j]` is the minimum of those 4 results.

### Code (Cleaned):

```java []
class Solution {
    public int maxProductPath(int[][] grid) {
        int m = grid.length, n = grid[0].length;
        long MOD = 1000000007;

        long[][] mx = new long[m][n];
        long[][] mn = new long[m][n];

        mx[0][0] = mn[0][0] = grid[0][0];

        // first row
        for (int j = 1; j < n; j++) {
            mx[0][j] = mn[0][j] = mx[0][j - 1] * grid[0][j];
        }

        // first column
        for (int i = 1; i < m; i++) {
            mx[i][0] = mn[i][0] = mx[i - 1][0] * grid[i][0];
        }

        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                long x = grid[i][j];

                long a = mx[i - 1][j] * x;
                long b = mn[i - 1][j] * x;
                long c = mx[i][j - 1] * x;
                long d = mn[i][j - 1] * x;

                mx[i][j] = Math.max(Math.max(a, b), Math.max(c, d));
                mn[i][j] = Math.min(Math.min(a, b), Math.min(c, d));
            }
        }

        long ans = mx[m - 1][n - 1];
        if (ans < 0) return -1;
        return (int)(ans % MOD);
    }
}
```

---

# ⏱️ Complexity Analysis

- We fill 2 DP tables of size `m x n`:
  - Each cell takes O(1) time.
- Time: `O(m * n)`
- Space: `O(m * n)` for the two DP arrays.

---

# 💡 Why Max & Min?

- When current cell is negative:
  - A **minimum** (most negative) product so far times this negative can become **maximum**.
  - A **maximum** positive times negative becomes a negative candidate for future flips.
- So we must carry both `mx` and `mn` forward at each step.

---

# 🔥 Comparison

| Approach         | Time      | Space     | Notes                                   |
|------------------|-----------|-----------|-----------------------------------------|
| DP with max/min  | O(m·n)    | O(m·n)    | Correctly handles negative & zero       |
| Simple max only  | Incorrect | -         | Fails when negatives flip the product   |

---

# 🧠 Takeaway

- In path/product DP with possible negative values, always track both `max` and `min` at each state.
- Only at the final cell do we check if the best product is non-negative and apply modulo.
