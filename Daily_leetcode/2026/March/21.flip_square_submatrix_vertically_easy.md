# 📌 Flip Square Submatrix Vertically

## 📜 Problem Statement

**Link:**  
[https://leetcode.com/problems/flip-square-submatrix-vertically/description/?envType=daily-question&envId=2026-03-21](https://leetcode.com/problems/flip-square-submatrix-vertically/description/?envType=daily-question&envId=2026-03-21)

You are given a 2D integer grid `grid`, and three integers `x`, `y`, and `k`:

- `(x, y)` is the **top-left** corner of a k x k **square** submatrix inside `grid`.
- You need to flip this k x k submatrix **vertically**:
  - Swap its first row with last row,
  - Second row with second-last row,
  - And so on.

Return the updated grid.

---

# 🧠 Understanding the Problem

- Only rows inside the k x k region `[x .. x + k - 1][y .. y + k - 1]` are affected.
- For each column `j` in `[y .. y + k - 1]`, and for each pair of rows from top and bottom inside the block, swap their values.
- Effectively a vertical reflection of that square block.

---

# 💻 Your Solution

### Approach:

1. Compute the bottom row index of the k x k block: `xEnd = x + k - 1`.
2. For each row `i` from `x` up to the halfway point `x + k/2` (integer division):
   - For each column `j` from `y` to `y + k - 1`:
     - Swap `grid[i][j]` and `grid[xEnd][j]`.
   - Decrement `xEnd` (move the bottom pointer one row up).
3. Return the modified `grid`.

### Code (Cleaned):

```java []
class Solution {
    public int[][] reverseSubmatrix(int[][] grid, int x, int y, int k) {
        int xEnd = x + k - 1;
        for (int i = x; i < x + k / 2; i++) {
            for (int j = y; j < y + k; j++) {
                int temp = grid[i][j];
                grid[i][j] = grid[xEnd][j];
                grid[xEnd][j] = temp;
            }
            xEnd--;
        }
        return grid;
    }
}
```

---

# ⏱️ Complexity Analysis

- Swaps are done for about `k/2` rows and `k` columns:

```
Time  → O(k²)
Space → O(1)  (in-place swaps)
```

---

# 💡 Key Insight

- Vertical flipping of a submatrix is just symmetric row swapping around its vertical center line.
- You never need extra storage beyond a temporary variable for swapping.

---

# 🧠 Takeaway

- For flipping submatrices, think in terms of two pointers (top and bottom rows) moving towards each other and swapping in place.
