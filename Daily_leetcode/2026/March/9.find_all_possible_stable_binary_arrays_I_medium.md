# 📌 Find All Possible Stable Binary Arrays I

## 📜 Problem Statement

**Link:**  
[https://leetcode.com/problems/find-all-possible-stable-binary-arrays-i/description/?envType=daily-question&envId=2026-03-09](https://leetcode.com/problems/find-all-possible-stable-binary-arrays-i/description/?envType=daily-question&envId=2026-03-09)

Given `zero` zeros and `one` ones, and a limit integer `limit`,  
Return the count of different binary arrays of length `zero + one` such that:
- Any segment of consecutive zeros has length at most `limit`.
- Any segment of consecutive ones has length at most `limit`.

**Modulo:** return answer mod 10⁹+7.

**Example:**  
Input: `zero = 2, one = 2, limit = 1`  
Output: `6`

---

# 🧠 Understanding the Problem

- Arrange zeros and ones such that there are **no more than `limit` consecutive 0s or 1s**.
- Count all such distinct arrays.
- DP state: How many zeros and ones remain, what was placed last (0/1), and how many consecutive of that element came before.

---

# 💻 Your Solution

### Approach:

- Use 3D DP: `dp[z][o][last]`
  - `z`: zeros left, `o`: ones left, `last`: 0 (ended with 0), 1 (ended with 1)
- Base cases:
  - When all zeros (and allowed), only one possible placement starting with zeros.
  - When all ones (and allowed), only one possible arrangement starting with ones.
- For each position:
  - To end with 0: place a 0 if allowed, and sum ways ending with either 0 or 1 previously (handle overlimit).
  - Symmetric for ending with 1.
- Subtract arrangements where over-limit runs are formed.

### Code (Cleaned):

```java []
class Solution {
    public int numberOfStableArrays(int zero, int one, int limit) {
        int MOD = 1000000007;
        long[][][] dp = new long[zero + 1][one + 1][2];
        // base cases: only zeros
        for (int i = 1; i <= Math.min(zero, limit); i++) {
            dp[i][0][0] = 1;
        }
        // base cases: only ones
        for (int j = 1; j <= Math.min(one, limit); j++) {
            dp[0][j][1] = 1;
        }
        for (int z = 1; z <= zero; z++) {
            for (int o = 1; o <= one; o++) {
                // place 0 at the end
                dp[z][o][0] = (dp[z - 1][o][0] + dp[z - 1][o][1]) % MOD;
                if (z - limit - 1 >= 0) {
                    dp[z][o][0] = (dp[z][o][0] - dp[z - limit - 1][o][1] + MOD) % MOD;
                }
                // place 1 at the end
                dp[z][o][1] = (dp[z][o - 1][0] + dp[z][o - 1][1]) % MOD;
                if (o - limit - 1 >= 0) {
                    dp[z][o][1] = (dp[z][o][1] - dp[z][o - limit - 1][0] + MOD) % MOD;
                }
            }
        }
        return (int) ((dp[zero][one][0] + dp[zero][one][1]) % MOD);
    }
}
```

---

# ⏱️ Complexity Analysis

- Time: O(zero·one)
- Space: O(zero·one) (for DP array)
- Everything is tabulated, so it's efficient for reasonable input limits.

---

# 💡 Key Insight

- Split the problem into: arrangements ending with 0 or 1.
- Count, then subtract any cases creating too-long runs using DP transitions.

# 🧠 Takeaway

- Stateful DP can elegantly count constructions with segment/runs limits.
- When both zeros and ones have maximum-in-run constraints, DP over remaining and last-used is a common standard approach.
