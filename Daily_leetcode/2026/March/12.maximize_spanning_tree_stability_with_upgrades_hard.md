# 📌 Maximize Spanning Tree Stability With Upgrades

## 📜 Problem Statement

**Link:**  
[https://leetcode.com/problems/maximize-spanning-tree-stability-with-upgrades/description/?envType=daily-question&envId=2026-03-12](https://leetcode.com/problems/maximize-spanning-tree-stability-with-upgrades/description/?envType=daily-question&envId=2026-03-12)

Given a graph with `n` nodes and `edges`, each edge described as `[u, v, s, must]`:
- `u`, `v`: endpoints
- `s`: initial "stability" of the edge (integer)
- `must`: 1 if the edge is mandatory (part of the spanning tree without upgrade), 0 if optional

You may "upgrade" up to `k` optional edges (not more than k), where upgrading edge doubles its stability.  
Find the **maximum stability X** so that you can build a spanning tree using all mandatory edges and upgraded/free optional edges, so that every edge included in the tree has stability at least `X`. Return `-1` if impossible.

---

# 🧠 Understanding the Problem

- You must use all mandatory edges (with at least stability X, and they can't be upgraded).
- Optional edges are free to use if their (possibly upgraded) stability ≥ X.
- You may use at most k upgrades (where doubling the stability is enough to cross X for weak edges).
- Need to check if a spanning tree including all nodes and all mandatory edges can be formed with at most k upgrades.

---

# 💻 Your Solution

### Approach:

- Binary search answer for X (maximum achievable stability).
- For each candidate X:
  1. Use DSU to unite all mandatory edges with stability ≥ X. If any fail (cycle or too weak), return false for this X.
  2. Add all (optional, no-upgrade) edges with stability ≥ X to the spanning candidate.
  3. Try to add as many (optional, upgradable) edges—edges with stability < X but 2·s ≥ X—using upgrades as needed (at most k upgrades).
- After all connections, check if all nodes are connected (components == 1) and upgrade budget not exceeded.

### Code (Cleaned):

```java []
class DSU {
    int[] parent, rank;
    int components;

    public DSU(int n) {
        parent = new int[n];
        rank = new int[n];
        components = n;
        for (int i = 0; i < n; i++) parent[i] = i;
    }

    public int find(int x) {
        return parent[x] == x ? x : (parent[x] = find(parent[x]));
    }
    public boolean unite(int a, int b) {
        int pa = find(a), pb = find(b);
        if (pa == pb) return false;
        if (rank[pa] < rank[pb]) { int t = pa; pa = pb; pb = t; }
        parent[pb] = pa;
        if (rank[pa] == rank[pb]) rank[pa]++;
        components--;
        return true;
    }
}

class Solution {
    public boolean canAchieve(int n, int[][] edges, int k, int x) {
        DSU dsu = new DSU(n);
        for (int[] e : edges)
            if (e[3] == 1) {
                if (e[2] < x || !dsu.unite(e[0], e[1])) return false;
            }
        for (int[] e : edges)
            if (e[3] == 0 && e[2] >= x)
                dsu.unite(e[0], e[1]);
        int usedUpgrades = 0;
        for (int[] e : edges) {
            if (e[3] == 0 && e[2] < x && 2 * e[2] >= x) {
                if (dsu.unite(e[0], e[1])) {
                    usedUpgrades++;
                    if (usedUpgrades > k) return false;
                }
            }
        }
        return dsu.components == 1;
    }

    public int maxStability(int n, int[][] edges, int k) {
        DSU dsu = new DSU(n);
        for (int[] e : edges) if (e[3] == 1) if (!dsu.unite(e[0], e[1])) return -1;

        int low = 1, high = 200000, ans = -1;
        while (low <= high) {
            int mid = low + (high - low) / 2;
            if (canAchieve(n, edges, k, mid)) {
                ans = mid;
                low = mid + 1;
            } else {
                high = mid - 1;
            }
        }
        return ans;
    }
}
```

---

# ⏱️ Complexity Analysis

- For each binary search step: O(E·α(N)) (essentially O(E)), E = edges, N = nodes.
- Binary search over [1, 2e5] (≈ 18 steps).
- Overall: O(E·log(maxStability))

---

# 💡 Key Insight

- Binary search on answer is possible if you can decide, for each X, if a spanning tree exists under the k-upgrade rule.
- DSU (Union-Find) makes checking connectivity/cycles efficient.

# 🧠 Takeaway

- For optimizing "properties" of MST under constraints, binary search + checked feasibility (often with DSU) is a powerful template.
- Always validate that required edges can participate before searching further.
