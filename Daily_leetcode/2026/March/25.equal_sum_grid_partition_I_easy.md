# 📌 Equal Sum Grid Partition I

## 📜 Problem Statement

**Link:**  
[https://leetcode.com/problems/equal-sum-grid-partition-i/description/?envType=daily-question&envId=2026-03-25](https://leetcode.com/problems/equal-sum-grid-partition-i/description/?envType=daily-question&envId=2026-03-25)

You are given a 2D integer grid `grid` of size `m x n`.

You want to check whether you can **partition the grid** using:
- either a **horizontal line** between two rows, or
- a **vertical line** between two columns,

such that the **sum of elements on one side equals the sum on the other side**.

Formally:
- There exists a row index `r` (0-based, 0 ≤ r < m-1) such that sum of rows `0..r` equals sum of rows `r+1..m-1`,  
  OR
- There exists a column index `c` (0-based, 0 ≤ c < n-1) such that sum of columns `0..c` equals sum of columns `c+1..n-1`.

Return `true` if such a partition exists, otherwise `false`.

---

# 🧠 Understanding the Problem

- Compute sums per row and per column.
- Then compute their prefix sums.
- For each possible partition between rows:
  - Check if prefix sum up to row `i` equals total - that prefix (sum of remaining rows).
- Same for columns.

If any row or any column splits the total sum evenly, the grid can be partitioned as asked.

---

# 💻 Your Solution

### Approach:

1. Build `xPresum` of length `m` for row sums.
2. Build `yPresum` of length `n` for column sums.
3. Prefix them so `xPresum[i]` is sum of rows `0..i`, `yPresum[j]` is sum of cols `0..j`.
4. Let `xTotal = xPresum[m-1]`, `yTotal = yPresum[n-1]` (both equal overall grid sum).
5. For each row index `i`:
   - If `xPresum[i] == xTotal - xPresum[i]`, return true.
6. For each column index `i`:
   - If `yPresum[i] == yTotal - yPresum[i]`, return true.
7. Else, return false.

### Code (Cleaned):

```java []
class Solution {
    public boolean canPartitionGrid(int[][] grid) {
        long[] xPresum = new long[grid.length];
        long[] yPresum = new long[grid[0].length];

        for (int i = 0; i < grid.length; i++) {
            for (int j = 0; j < grid[0].length; j++) {
                xPresum[i] += grid[i][j];
                yPresum[j] += grid[i][j];
            }
        }

        for (int i = 1; i < xPresum.length; i++) {
            xPresum[i] += xPresum[i - 1];
        }
        for (int i = 1; i < yPresum.length; i++) {
            yPresum[i] += yPresum[i - 1];
        }

        long xTotal = xPresum[xPresum.length - 1];
        long yTotal = yPresum[yPresum.length - 1];

        for (int i = 0; i < xPresum.length; i++) {
            if (xPresum[i] == xTotal - xPresum[i]) {
                return true;
            }
        }

        for (int i = 0; i < yPresum.length; i++) {
            if (yPresum[i] == yTotal - yPresum[i]) {
                return true;
            }
        }

        return false;
    }
}
```

---

# ⏱️ Complexity Analysis

- Single pass over grid to build row/column sums: O(m·n)
- Prefix sums over rows and columns: O(m + n)
- Checking row and column partitions: O(m + n)

Overall:

```
Time  → O(m·n)
Space → O(m + n)
```

---

# 💡 Key Insight

- Row partition reduces to splitting 1D array of row sums into two equal parts via prefix sum.
- Same for columns.
- If either 1D prefix sum array has a point where prefix = total - prefix, you have a valid partition.

---

# 🔥 Comparison

| Approach            | Time    | Space    | Notes                               |
|---------------------|---------|----------|-------------------------------------|
| Row+Column prefix   | O(m·n)  | O(m + n) | Clean, optimal for constraints      |
| Brute recompute sum | O(m·n·(m + n)) | O(1) | Too slow for larger grids         |

---

# 🧠 Takeaway

- Grid partition/split questions often reduce to prefix-sum checks on row/column aggregates.
- Always compute row/column sums first, then reason in 1D.
