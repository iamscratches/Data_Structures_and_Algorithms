# 📌 Count Submatrices With Top-Left Element and Sum Less Than k

## 📜 Problem Statement

**Link:**  
[https://leetcode.com/problems/count-submatrices-with-top-left-element-and-sum-less-than-k/description/?envType=daily-question&envId=2026-03-18](https://leetcode.com/problems/count-submatrices-with-top-left-element-and-sum-less_than-k/description/?envType=daily-question&envId=2026-03-18)

Given an integer matrix `grid` and integer `k`,  
Count the number of submatrices for which:
- The submatrix starts at the top-left element `(0,0)`
- The sum of all elements in the submatrix is less than or equal to `k`.

---

# 🧠 Understanding the Problem

- All submatrices must be anchored at the top-left corner (starts (0,0)).
- Each such submatrix is defined by bottom-right coordinates `(i,j)`.
- Need to quickly compute sum for all `(0,0)` to `(i,j)` submatrices, and count those ≤ k.

---

# 💻 Your Solution

### Approach:

- 2D prefix sum: grid[i][j] = sum of submatrix from (0,0) to (i,j).
- Precompute first row and first column accumulations with early stopping if exceeds k.
- For other entries, compute grid[i][j] in-place:
    - grid[i][j] += grid[i-1][j] + grid[i][j-1] - grid[i-1][j-1]
- At each (i, j), if sum ≤ k, increment count; else, break early for the row.

### Code (Cleaned):

```java []
class Solution {
    public int countSubmatrices(int[][] grid, int k) {
        if (grid[0][0] > k) return 0;
        int count = 1, row = 1, col = 1;
        while (col < grid[0].length && grid[0][col] + grid[0][col - 1] <= k) {
            grid[0][col] += grid[0][col - 1];
            count++; col++;
        }
        while (row < grid.length && grid[row][0] + grid[row - 1][0] <= k) {
            grid[row][0] += grid[row - 1][0];
            count++; row++;
        }
        for (int i = 1; i < row; i++) {
            for (int j = 1; j < col; j++) {
                grid[i][j] += grid[i - 1][j] + grid[i][j - 1] - grid[i - 1][j - 1];
                if (grid[i][j] <= k) count++;
                else break;
            }
        }
        return count;
    }
}
```

---

# ⏱️ Complexity Analysis

- First row and col: O(m + n)
- Main 2D scan: up to min(m, n)² for small valid region (early exit).
- Space: O(1) extra, grid modified in-place.

---

# 💡 Key Insight

- Use prefix sums for O(1) submatrix queries, counting as we build.
- Break early in row/col if sum constraint fails, since all further submatrices will exceed k.

# 🧠 Takeaway

- For anchored submatrix sum problems, in-place prefix sum with early stopping is both time- and space-efficient.
- Leverage matrix structure to quickly cut off impossible submatrices.
