# 📌 Construct Product Matrix

## 📜 Problem Statement

**Link:**  
[https://leetcode.com/problems/construct-product-matrix/description/?envType=daily-question&envId=2026-03-24](https://leetcode.com/problems/construct-product-matrix/description/?envType=daily-question&envId=2026-03-24)

You are given an `m x n` matrix `grid` and a modulus `12345`.

You need to construct another `m x n` matrix `ans` such that:

- For each cell `(i, j)`, `ans[i][j]` equals the product of **all elements in `grid` except** `grid[i][j]`, taken modulo `12345`.

Effectively, each `ans[i][j]` is:

\[
\left(\prod_{\substack{(x,y)\neq(i,j)}} grid[x][y]\right) \bmod 12345
\]

---

# 🧠 Understanding the Problem

- This is a 2D version of the classic "product of array except self" problem.
- Direct computation per cell is O((mn)²) — too slow.
- Better approach:
  - Flatten the grid into a 1D array.
  - Build prefix products and suffix products.
  - For each index i, result[i] = prefix[i] × suffix[i] mod 12345.
  - Map back the 1D result into 2D matrix.

---

# 💻 Your Solution

### Approach:

1. Flatten the `m x n` grid into a 1D array `arr` of size `m*n`.
2. Build:
   - `pre[i]` = product of all elements before `i` (0..i-1), modulo 12345.
   - `suf[i]` = product of all elements after `i` (i+1..end), modulo 12345.
3. For each index `i`, `res[i] = pre[i] * suf[i] mod 12345` — product of all except `arr[i]`.
4. Reconstruct the answer matrix `ans` from `res` in row-major order.

### Code (Cleaned):

```java []
class Solution {
    public int[][] constructProductMatrix(int[][] grid) {
        int m = grid.length, n = grid[0].length;
        int size = m * n;
        int[] arr = new int[size];

        int idx = 0;
        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++)
                arr[idx++] = grid[i][j];

        int[] pre = new int[size];
        int[] suf = new int[size];
        Arrays.fill(pre, 1);
        Arrays.fill(suf, 1);

        for (int i = 1; i < size; i++)
            pre[i] = (int)((1L * pre[i - 1] * arr[i - 1]) % 12345);

        for (int i = size - 2; i >= 0; i--)
            suf[i] = (int)((1L * arr[i + 1] * suf[i + 1]) % 12345);

        int[] res = new int[size];
        for (int i = 0; i < size; i++)
            res[i] = (int)((1L * pre[i] * suf[i]) % 12345);

        int[][] ans = new int[m][n];
        idx = 0;
        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++)
                ans[i][j] = res[idx++];

        return ans;
    }
}
```

---

# ⏱️ Complexity Analysis

- Flatten grid: O(mn)
- Prefix and suffix products: O(mn)
- Construct result and rebuild matrix: O(mn)

Overall:

```
Time  → O(mn)
Space → O(mn) for flattened arrays and prefix/suffix
```

---

# 💡 Key Insight

- The 2D matrix can be treated as a 1D array in row-major order.
- Prefix/suffix product trick avoids division and handles zeros cleanly when working purely multiplicatively under modulo.

---

# 🔥 Comparison

| Approach                      | Time       | Space     | Notes                             |
|-------------------------------|------------|-----------|-----------------------------------|
| Naive per-cell full product   | O((mn)²)   | O(1)      | Too slow                          |
| Prefix–suffix on flattened    | O(mn)      | O(mn)     | Clean, scalable, no division used |

---

# 🧠 Takeaway

- Many 2D product/sum problems can be reduced to classic 1D patterns by flattening.
- Prefix/suffix arrays are powerful for “product/sum of all except self” without division.
