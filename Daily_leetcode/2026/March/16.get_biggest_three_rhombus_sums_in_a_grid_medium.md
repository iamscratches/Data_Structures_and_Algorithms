# 📌 Get Biggest Three Rhombus Sums in a Grid

## 📜 Problem Statement

**Link:**  
[https://leetcode.com/problems/get-biggest-three-rhombus-sums-in-a-grid/description/?envType=daily-question&envId=2026-03-16](https://leetcode.com/problems/get-biggest-three-rhombus-sums-in-a-grid/description/?envType=daily-question&envId=2026-03-16)

You are given an `m x n` integer grid.  
A "rhombus" in the grid is a diamond-shaped set of cells of a certain size.
- A rhombus of size 0 is just a single cell (center).
- A rhombus of size k>0 looks like a square with corners on diagonals around center (i, j).

Find the biggest three **distinct** rhombus sums in the grid.

---

# 🧠 Understanding the Problem

- For every cell, with increasing radius (size), try to form a rhombus and sum its border.
- Take diagonal "walks" from (i, j) outward for four directions, summing.
- Use a TreeSet to collect unique sums.

---

# 💻 Your Solution

### Approach:

1. For every cell (i, j) consider it as rhombus center.
2. Add single cell value as a rhombus (size 0).
3. For size s = 1 up to possible, walk diagonals from (i, j) to form rhombus and sum the perimeter:
   - Move s up-right, s down-right, s down-left, s up-left along border.
   - Take care to not double-count corners.
4. Store all generated sums in a TreeSet for uniqueness.
5. Output the largest three unique sums.

### Code (Cleaned):

```java []
class Solution {
    public int[] getBiggestThree(int[][] grid) {
        int m = grid.length;
        int n = grid[0].length;
        TreeSet<Integer> sums = new TreeSet<>();

        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                sums.add(grid[i][j]);
                for (int s = 1; i + 2 * s < m && j - s >= 0 && j + s < n; s++) {
                    int currentSum = 0;
                    for (int k = 0; k < s; k++) currentSum += grid[i + k][j + k];
                    for (int k = 0; k < s; k++) currentSum += grid[i + s + k][j + s - k];
                    for (int k = 0; k < s; k++) currentSum += grid[i + 2 * s - k][j - k];
                    for (int k = 0; k < s; k++) currentSum += grid[i + s - k][j - s + k];
                    sums.add(currentSum);
                }
            }
        }

        int count = Math.min(sums.size(), 3);
        int[] result = new int[count];
        for (int i = 0; i < count; i++) {
            result[i] = sums.pollLast();
        }
        return result;
    }
}
```

---

# ⏱️ Complexity Analysis

- For every cell (m·n), sizes O(min(m, n)), every border O(size).
- Worst-case O(m·n·min(m, n)), but small grid bounds (m, n ≤ 50).

---

# 💡 Key Insight

- TreeSet keeps sums unique and sorted for easy top k extraction.
- Only border cells are summed for each rhombus (not all inside cells).

---

# 🔥 Comparison

| Approach    | Time                 | Notes                            |
|-------------|----------------------|----------------------------------|
| Brute force | O(m·n·min(m, n))     | Works due to small grid limits   |
| Fastest     | O(m·n·s)             | Easy, needs careful bounds check |

---

# 🧠 Takeaway

- When a small grid and geometric substructure, brute-force with deduplication is often fastest and most understandable.
- Use collections like TreeSet to get unique top-K values efficiently.
